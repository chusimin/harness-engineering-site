01 · IT IS WHAT

它是什么

三个尺度下的统一定义。Agent = Model + Harness。

Agent = Model + Harness

Harness（驾驭套件）= 包在模型外面、把被动模型变成能干活 Agent 的那一整圈代码与结构。

02 · LINEAGE

它从哪来

从 prompt engineering 到 harness engineering，每一次命名升级都是因为"控制面积"不够了。

2022
Prompt Engineering
一切起点 —— 研究如何写好提示词，让模型给出更好的答案。
2023
Agent 框架涌现
LangChain、AutoGPT、BabyAGI 等框架引入 Agent 概念，工具链爆发。
2024
Context Engineering
认识到"给模型什么上下文"比"怎么写提示词"更重要。控制面扩大。
2025 — NOW
Harness Engineering
Anthropic、OpenAI 先后发文正式命名。最外层、最完整的实践框架。

提示词工程 ⊂ 上下文工程 ⊂ Harness 工程 —— 嵌套关系，每一层比上一层管得更多。

03 · ANATOMY

解剖学

拆开一个 Harness，看里面每个部件的职责与常见模式。

⟳
主循环 Loop
ReAct / Plan-and-Execute / 自适应策略选择。决定 Agent 每一步做什么。
◉
上下文管理
窗口裁剪、摘要压缩、RAG 注入、会话记忆。Agent 的注意力分配。
⚡
工具与执行
Tool Schema 定义、权限边界、沙箱隔离、重试与降级。
◈
评测与可观测
离线评测集、在线 A/B、trace 链路、token 成本监控。
🛡
安全围栏
输入过滤、输出审查、人机交接条件、权限升级流程。
⚙
编排层
多 Agent 协作、任务分解与分发、状态持久化、错误恢复。

05 · METHODOLOGY

怎么做

从 Anthropic 和 OpenAI 一手文档中提炼的实操指南 —— PM 能上手的事。

1
定义 Agent 边界
哪些任务交给 Agent、哪些留给人类、什么时候人机交接。
2
设计工具集
给 Agent 什么工具、工具 schema 怎么写、权限边界画在哪。
3
构建评测体系
离线评测集、关键指标、A/B 实验框架、回归检测。
