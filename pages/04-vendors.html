厂商全景。

三家旗手、三条赛道、三种哲学 —— 不是谁更强,是赛道根本不止一条。

20 分钟阅读 7 个小节 6 个可视化
S1 · 三家、三条赛道

三家不在同一根赛道上竞速。

板块② 讲过这个词怎么火起来的,板块③ 解剖了组件级 harness 的器官。 这一板块,把镜头拉远 —— 看真实世界三家头部公司,各自把 harness 做成了什么样。

S2 · OpenAI

五个月、零行手写代码
的疯狂实验。

OpenAI 内部启动了一个实验: 用 AI 从零写一个真实的软件产品,全程不允许工程师手写一行代码。 但一开始进展很不顺利 —— 问题不是"模型不够聪明", 而是 agent 经常走错方向、重复犯同样的错误。 他们意识到:真正的功夫,在于把 Harness 设计好。

PM 落点 —— ❶ 环境比模型更重要:实验一开始失败不是模型问题,是环境没搭好。 ❷ 仓库 / 唯一事实来源是基础设施。 ❸ 自验证闭环 = 准确率的真正杠杆。 ❹ 预防 + 清扫两手都要。
S3 · Anthropic

Planner / Generator / Evaluator
—— 多 agent 合力。

Anthropic 做了个实验:让 agent 去克隆 claude.ai。 最初只用一个 agent,结果惨败 —— 想一口气做完所有功能,上下文满了,抛下烂摊子,下一个 agent 接手完全不知道发生了啥。 一个 agent 撑不了那么长的任务。

一个时间维度上的细节:Anthropic 一开始在提示词里强制 Generator"一次只做一个功能点"。 后来 Opus 4.6 出来,这条约束被去掉了 —— 模型够强,能自己统筹。 Harness 是刚好够用,模型变强,harness 该精简。
PM 落点 —— ❶ 任务超出单 agent 范围 → 上任务级编排。 ❷ 独立 Evaluator 是"王婆卖瓜"的解药。 ❸ Full Harness 有成本 —— PM 要看 ROI 值不值。 ❹ Harness 是动态的 —— 模型升级,你的 harness 可能要减,不是加。
S4 · Cursor

没有自家模型,
把组件级做到极致。

Cursor 是一个 IDE(由 VSCode fork 而来)。 它没有自家前沿模型 —— 用 Claude、GPT、Gemini。 它甚至不做通用 AI 助手 —— 只做一件事:让工程师在编辑器里写代码。 就这一个聚焦,两年做到最受欢迎的 AI IDE 之一。

PM 落点 —— ❶ 一个场景的极致,胜过十个场景的平庸。 ❷ "无意识好用"(speed)是真护城河 —— 用户迷恋"零等待"。 ❸ 模型路由是成本结构,不是技术细节 —— 没有这个,规模化注定亏钱。 ❹ 没有模型也能赢 —— 只要你的 harness 比对手好。

S6 · 选型地图

你的产品
像哪一家？

对号入座的本质,是先识别你产品的"尺度" —— 你包的是一次模型调用 / 一个长任务 / 一支队伍? 尺度决定 harness 的形态。

S7 · 一句话总结
三家、三种哲学、三个尺度,各占一格 —— OpenAI 用组织级证明"AI 能经营产品"; Anthropic 用任务级证明"多 agent 合力 > 单 agent 死撑"; Cursor 用组件级证明"做最好的 harness,可以打败拥有模型的人"。 它们不是竞速,是把 AI 产品的设计空间撑开成了三条赛道。
