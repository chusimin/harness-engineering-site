§1 · 被低估的真相

为什么产品差这么远？

2025 年起，AI 产品的胜负手悄悄变了。 过去问的是"谁的模型更强"，现在三个截然不同的产品，背后可能是同一个模型。

§2 · 一句话定义

Harness = 那一整圈代码。

一个 agent 从来不是"一个很强的模型"。它是 —— 一个模型，加上一套把模型驾驭起来的代码。

它不是什么 —— ❶ 不是模型本身（模型是引擎，harness 是底盘）。 ❷ 不只是提示词（提示词只是 harness 最里面的一层）。 ❸ 不是某个框架（LangGraph、Claude Agent SDK 这些是"用来搭 harness 的工具"，不等于 harness 本身）。

§5 · 尺度

它有多大？
三个尺度，套着的。

这是板块① 最重要的一个框架。 Harness 不是固定大小的东西 —— 它有三个尺度，一个套一个。 各家厂商定义看起来打架，其实只是站在这把"尺度尺"的不同刻度上。

Harness 不是越大越好，是"刚好够用"。 模型一升级，本来需要任务级编排的活，可能塌回组件级一个 agent 就能干。 —— Anthropic 在官方文章里点出的关键。

§7 · 一句话总结
Harness 是包在模型外面的那一整圈代码与结构， 它把被动的模型变成能干活的 agent； 它的构成可记成「转 · 知 · 做 · 检」； 它有「组件 / 任务 / 组织」三个尺度； 各家定义看似打架，其实只是站在这把"尺度尺"的不同刻度上。
