02 · GLOSSARY

35 个术语速查

PM 跟工程师讨论 harness 前，把这张表过一遍。同一套术语，才能同一张图。

Agent
Model + Harness 的总和；能感知、推理、行动、自我修正的程序。
Harness
包在模型外面让它能干活的那一整圈代码与结构。
Loop
Agent 的主循环：感知 → 推理 → 行动 → 观察 → 再决策。
Tool
Agent 可调用的外部能力（搜索、写文件、调 API 等）。
Tool Schema
工具的"说明书"，含名字、描述、参数类型、返回结构。
Context Window
模型一次能看到的 token 上限；超过要截断或压缩。
Memory
跨 turn / 跨 session 的信息存储；分短期 / 长期 / 情景。
RAG
Retrieval-Augmented Generation；先检索再生成的范式。
Embedding
把文本/图像压缩成定长向量，便于相似度检索。
Vector Store
存 embedding 的数据库，支持近邻检索（如 FAISS、Pinecone）。
ReAct
Reasoning + Acting 范式，让模型边推理边调工具。
Plan-and-Execute
先一次性产出完整计划，再分步执行的范式。
Reflection
让模型回顾自己的输出并打分、改进的机制。
Self-Critique
模型对自己回答打分并指出问题，再修正的子模式。
Eval Set
评测集；用来衡量 agent 能力的标注样本集合。
A/B Test
在线流量分两组对比新旧版本，看核心指标差异。
Trace
一次完整调用的链路记录；含所有 span 与中间状态。
Span
trace 的单个步骤，比如一次模型调用或一次工具执行。
Token
模型输入/输出的最小计费单位，约等于 0.7 个英文词。
Budget
单任务/单租户的 token 或 $ 上限，超出熔断或叫人。
Sandbox
隔离的执行环境（容器/VM），限制 agent 能动的资源。
Permission Boundary
权限边界；agent 能读/写/执行的资源白名单。
Human-in-the-Loop
关键节点让人确认或接管的设计模式。
Handoff
Agent 到人、或 agent 到 agent 的任务交接。
Checkpoint
把中间状态持久化，失败时可从此点重启。
Sub-Agent
主 agent 启动的子任务专员；用完即销毁。
Orchestration
编排；决定多 agent / 多工具的协作流程。
Workflow
结构化的多步流程；可固定也可动态。
State Machine
状态机；agent 当前在哪个状态、能转到哪些状态。
MCP
Model Context Protocol；Anthropic 提出的工具/上下文标准协议。
Tool Use
模型决定何时调哪个工具、传什么参数的能力。
Function Calling
OpenAI 的 tool use 实现；模型输出 JSON 调用函数。
Structured Output
强制模型按 JSON Schema 输出，便于下游解析。
Streaming
边生成边返回 token；改善延迟感与可中断性。
Caching
prompt prefix / response 缓存；省 token、降延迟。
Prompt Engineering
研究怎么写一条 prompt；harness 的最内层。
Context Engineering
研究"给模型什么"；含 RAG、记忆、工具描述。

03 · INTERVIEW

AIPM 面试题 · 10 道

来自一线 AI 公司的真实题库 —— 概念题、设计题、案例题混搭。看的不是答案，是思考路径。

QUESTION · 01
prompt engineering 和 harness engineering 有什么区别？
考察点：嵌套关系认知。是否能说出"prompt ⊂ context ⊂ harness"以及为什么外扩。
QUESTION · 02
给一个 coding agent 设计评测指标，你会选哪 5 个？
考察点：评测体系完整性。任务完成率 + 成本 + 轮次 + 介入率 + 安全事件率。
QUESTION · 03
你的 agent 在线指标突然下降 10%，怎么排查？
考察点：事故响应流程。看 trace 共性 → 看是否近期变更 → 灰度回滚 → 加入回归集。
QUESTION · 04
怎么决定一个 Agent 该用 ReAct 还是 Plan-and-Execute？
考察点：架构选型。任务可预测 → Plan；环境动态 → ReAct；通常混合。
QUESTION · 05
设计一个工具的 schema，PM 要审查哪些项？
考察点：工程素养。输入输出/错误/幂等/权限/可观测 五项 checklist。
QUESTION · 06
长任务 agent 跑到一半失败了，你的恢复策略是什么？
考察点：checkpoint 设计。状态序列化 → 断点续跑 → 失败原因分类 → 重试 vs 升级人工。
QUESTION · 07
什么场景下你会建议"这个不要做成 agent"？
考察点：边界判断。强主观、不可逆、高合规、低样本、要求人际信任 五大反信号。
QUESTION · 08
怎么平衡 agent 自主性与安全性？
考察点：权限边界设计。最小权限默认 + 高风险动作显式 opt-in + 审计日志 + 人机交接条件。
QUESTION · 09
prompt caching 能省成本，PM 决策时怎么权衡？
考察点：工程经济学。命中率 / TTL / prompt 结构 → 缓存命中 → 单任务成本下降百分比。
QUESTION · 10
MCP 协议出现后，你的产品要不要适配？为什么？
考察点：趋势判断。MCP 是工具/上下文统一协议 → 长期降低集成成本 → 现状成熟度 → 投入产出比。

04 · CAREER MAP

AIPM 就业地图

2026 年 AI PM 三条主路径，薪资范围参考一线市场（北京/上海/深圳/硅谷）。

TRACK · 01
Vertical AI PM · 行业纵深

做特定领域的 agent —— 法律、医疗、金融、电商、客服、教育。深耕一个垂直，懂业务比懂技术更值钱。

典型岗位
Legal AI PM / Medical AI PM / EC AI PM
需要技能
行业 know-how、用户画像、合规、评测体系搭建
薪资范围
25 - 60k · 月
TRACK · 02
Platform AI PM · Harness / 基建

做 agent 平台、工具链、可观测设施。给一线业务 PM 提供"乐高积木"，是真正的 harness engineer。

典型岗位
Agent Platform PM / MLOps PM / Infra PM
需要技能
工程深度、API 设计、SDK 抽象、平台经济学
薪资范围
35 - 80k · 月
TRACK · 03
Eval & Safety PM · 评测与安全

做评测集、红队测试、安全围栏、合规审查。监管收紧后的稀缺岗位，跟法务/政策深度协作。

典型岗位
Eval PM / Trust & Safety PM / AI Policy PM
需要技能
统计学、对抗性测试、监管理解、跨职能沟通
薪资范围
30 - 70k · 月

招聘经理看什么信号

一线 AI 公司面 AIPM 时，下面 3 个信号胜过任何头衔：

  • 具备工程语言：能跟工程师讨论上下文窗口、tool schema、retry 策略 —— 不是写代码，是用同一套术语
  • 能读懂 trace：给你一段 trace 日志，能从中读出"agent 在第几轮卡住、为什么卡、该改 prompt 还是改工具"
  • 能定义评测：给一个新场景，能在 30 分钟内写出 10 条评测样本 + 3 个量化指标 + 1 个 baseline

参考： OpenAI Codex 招聘文章 明确说出 PM 候选人要有"工程同理心"和"评测思维"，比传统 PRD 撰写能力优先级更高。
