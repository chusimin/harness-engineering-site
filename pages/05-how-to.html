§1 · 第一步

做 agent 产品的第一件事
不是选模型，是选尺度。

板块① 的"三个尺度"在这里第一次发挥用处。 选错尺度 = 白干。问 3 个问题就能定位 ——

🧩
COMPONENT
组件级

1 个 agent + 用户每次喊 + 单次任务

IDE 里的代码补全 / 嵌在 Office 的写作伴侣
🔄
TASK
任务级

多个 agent + 有终点的长任务

一句话造一个完整应用 / 一晚上整理一本书
🏢
ORGANIZATION
组织级

一支队伍 + 永远在演进的产品

公司内部 AI 编码平台 / agent 承包 80% 日常
选错尺度的代价 —— 用任务级 harness 解决一个组件级问题 = 用航母接客人。慢、贵、复杂度爆炸。
§2 · 分工

平台层 vs 应用层
你的活在应用层。

一个 harness 分两层。平台层已经有人做了，你不必造轮子。 认清"你能造什么"是方法论第一课。

最小 Harness 铁律
6 件全有，才叫 harness；
少一件，你做的是 demo。
§3 · 搭建顺序

6 个零件，
按什么顺序搭？

板块③ 讲了 6 个零件是什么，这里讲搭哪个先。 不要 6 件一起上 —— 一步一步来。每完成一步，你就有个"能跑的中间版本"。

§4 · Skills

CLAUDE.md / Skills / Tools
三件怎么分？

这三个名字经常搞混。它们不是同一层的东西 —— 把公司类比说清楚，一秒钟就懂。

§5 · 原则

做 harness 的
8 条核心原则。

把板块 ①-④ 散布的核心金句汇成一张清单。 任何一条违反，都会让你的 agent 产品结结实实栽跟头。

§6 · 反模式

别踩这些坑。
9 个反模式速查。

按"症状 / 怎么错的 / 怎么改"列。 每张卡片是一个真实的坑 —— 红色是错法，绿色是改法。

§7 · 迭代

搭完不是终点。
转起来才是。

Harness 不是交付一次就不动了。它是永远在转的循环 —— 搭、测、看数据、改、再搭。

§8 · Checklist

上线前 / 切模型前
必须过的两套 Checklist。

任何一项空着 = 裸奔上线。 切模型前的金句 —— "敢切，是因为能撤。"

§9 · 三步法

整套方法论
压成一句话。

下次你看任何一个 agent 产品的事故， 几乎都能定位到其中一步没做好。

§10 · 一句话总结
板块⑤ = 板块 ①-④ 的方法论收口。 做 agent 产品的第一件事不是选模型，是选尺度； 然后认清你的活在应用层那 3-4 格（记忆 / 边界 / 评测）； 按 MVP 顺序一件件搭； 上线前过 7 件套，切模型前过 4 件套。 8 条原则 + 9 条反模式背下来，你看任何 agent 产品事故都能秒定位。
