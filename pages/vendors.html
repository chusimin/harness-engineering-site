01 · OVERVIEW

五款主流 Coding Agent

截至 2026 年初，开发者圈子里被频繁讨论的 5 款"能真干活"的 coding agent。同样的目标，五种 harness 答卷。

终端原生的长程编码 Agent。以"sub-agent + plan mode + 长时任务"为最大特色。

CX

02 · VENDOR 01

Claude Code

Anthropic 官方的 CLI 编码 Agent。把"在你 repo 里干活"做到极致 —— 终端跑、长任务、可中断、可恢复。

Harness 设计特色

  • 长程任务 harness —— 单次任务可跑数十分钟到数小时，靠 checkpoint 和压缩上下文撑住长循环。
  • Sub-agent 派工 —— 主 Agent 把子任务派给独立 sub-agent，各自带干净 context，结果回主线。
  • Plan mode —— 先和你对齐计划再开干，避免一上来就乱改文件。
  • Skill / 命令体系 —— 用 SKILL.md 写"专家模块"，按需触发，等于把领域知识注成 sidecar。

选型场景

  • 项目级重构、跨文件大改
  • 需要在终端/服务器里跑、不接 IDE
  • 对 PR 形态结果不感兴趣，希望"直接改我本地 repo"
  • 团队已经吃 Anthropic 模型生态（API、Bedrock）

03 · VENDOR 02

Codex

OpenAI 的云端 coding agent。一句话给任务，云上沙箱开容器干活，结果以 PR 形态回到你 GitHub。

Harness 设计特色

  • 云端隔离沙箱 —— 每个任务跑在干净的容器里，Agent 在沙箱内有 root 权限、随便折腾。
  • 并行 fan-out —— 同一个需求可以同时派给多个 Agent，每个出一个候选 PR，你选最好的。
  • PR-style 输出 —— 不直接改你本地，而是输出 diff / branch / PR，强制走 code review 流程。
  • "agent-first" 默认 —— 设计上预设"由 Agent 主驱"，人参与的是 review 不是 driver。

选型场景

  • 有 GitHub workflow、习惯走 PR review
  • 不放心让 Agent 直接动本地，要隔离
  • 想同时跑多个候选方案做对比
  • 愿意按 cloud GPU/CPU 付费换隔离

04 · VENDOR 03

Cursor

Anysphere 出品的 VS Code fork。把 AI 从"右下角侧边栏"做进 IDE 主舞台 —— inline edit、Composer、多模型可选。

Harness 设计特色

  • Composer / Agent 模式 —— 多文件改写的"作曲模式"，一段需求横扫整个 repo。
  • 代码库索引 —— 后台把整库 embed 入向量库，回答前先 RAG。
  • 多模型路由 —— Claude、GPT、Gemini 任选；不同任务可挂不同模型。
  • Tab autocomplete —— 极低延迟的预测式补全，IDE 体验主线。

选型场景

  • 已经用 VS Code、不想换工作台
  • 需要在 IDE 里"边看边改"，不是把活全甩给 Agent
  • 想自由切换模型供应商，避免单一锁定
  • 个人开发者 / 小团队，按月订阅可承受
FORM IDE (VS Code fork) MODEL Claude / GPT / Gemini LOOP ReAct + Composer FOUNDED Anysphere · 2022

产品： Cursor 官网 · Agent 文档

05 · VENDOR 04

Windsurf

Codeium 推出的 IDE Agent（也是 VS Code fork）。差异点是 Cascade：链式工作流 + 跨步骤的"flow state"记忆。

Harness 设计特色

  • Cascade（级联工作流） —— 把多步改动串成一条链路，AI 主动接管下一步，不需要每步点确认。
  • Flow state 记忆 —— 跨步骤维持上下文与意图，包括"为什么改这里"。
  • Agent / Chat 双模 —— 想全自动选 Agent，想保持手动选 Chat。
  • 企业版自托管 —— Codeium 历来强调企业自部署、本地不出网。

选型场景

  • 长 session 编辑，希望少点确认按钮
  • 需要 Agent 接住"上一步是什么、为什么"
  • 对企业部署、数据不出网有硬要求
  • 从 Codeium 老用户升级
FORM IDE (VS Code fork) MODEL 多模型 + 自研 LOOP Cascade chain FOUNDED Codeium · 2021

产品： Windsurf 官网 · Cascade 文档

06 · VENDOR 05

Devin

Cognition AI 发布的"自主软件工程师"。给 Agent 一整套虚拟工位 —— shell + 浏览器 + 编辑器 + 长程规划。

Harness 设计特色

  • Full virtual workstation —— shell + 浏览器 + 文件管理 + 编辑器，Agent 像真实工程师那样开网页查文档。
  • 长程 plan + 重规划 —— 自动拆任务、跑得不顺会主动改 plan。
  • Slack / Linear 接入 —— 不只在 IDE 里，可以被你在协作工具里 @ 出来。
  • Session 录屏 —— 任务可回看像视频；可暂停、修正、续跑。

选型场景

  • 想用 "AI 同事" 范式，不只是 IDE 助手
  • 任务要查外部资料 / 调外部服务
  • 团队接 Slack / Linear，希望工具内 @ Agent
  • 容忍高单价、换"端到端自主"
FORM Cloud workstation MODEL 多模型组合 LOOP Plan + Replan FOUNDED Cognition · 2023
