01 · WHAT IS IT

是什么

Harness Engineering（驾驭工程）= 包在模型外面、把被动模型变成能干活 Agent 的那一整圈代码与结构。

01 · DEFINITION

一句话讲清楚

Agent = Model + Harness。模型负责"想"，harness 负责让它"能干活"。

Agent = Model + Harness

这个公式的两位作者 —— Anthropic OpenAI —— 在 2026 年初先后撰文，正式定义了这个工程领域。

模型本身是无状态的、被动的、不能调用工具的。而一个能干活的 Agent，需要状态管理、工具调用、安全围栏、可观测性、评测体系 …… 这些"包在模型外面的代码与结构"，就是 harness。

02 · THREE SCALES

三个尺度

同心圆模型 —— 各家定义不冲突，只是站在"尺度尺"的不同刻度。

COMPONENT · 组件级
裹一次模型调用

最小单位。包括上下文构造、工具调用、输出解析、重试、单条 trace。一个 harness 组件 = "一次有围栏的模型调用"。

代表观点： LangChain

TASK · 任务级
编排多 Agent 干成一件事

中等粒度。多个组件协作完成一个有终点的任务。包括分工、状态机、人机交接、最终交付物。

代表观点： Thoughtworks

ORG · 组织级
罩一支 Agent 队伍

最大粒度。一个长期运行的产品 + 一群 Agent。包括灰度发布、A/B 实验、评测集、成本治理、安全合规。

代表观点： Anthropic

组件级 ⊂ 任务级 ⊂ 组织级。 外层包裹内层 —— 你做一个组织级 harness，必然也在做任务级、组件级。

04 · CORE COMPONENTS

四大组件

一个 harness 不管多大，都可拆成这四个部件：转 · 知 · 做 · 检。

⟳
转 · LOOP

主循环：感知 → 推理 → 行动。Agent 的心跳。决定下一步做什么、何时停。

◉
知 · MEMORY

上下文管理：短期工作记忆 + 长期知识库。决定 Agent 知道什么、记得什么。

⚡
做 · TOOLS

工具与执行：可调用的工具集 + 沙箱与权限。决定 Agent 能动什么、不能动什么。

◈
检 · EVAL

评测与可观测：trace 链路、离线评测、在线 A/B。决定 Agent 跑得好不好。

05 · NESTED LAYERS

嵌套关系

Harness 工程不是凭空出现的，它是从更小的实践一层层包出来的。

最内层
提示词工程 · Prompt Engineering
研究"怎么写好一条 prompt"
⊂
中间层
上下文工程 · Context Engineering
研究"给模型什么上下文"（含 RAG、记忆、工具）
⊂
最外层
Harness 工程 · Harness Engineering
研究"包出一个能跑、能管、能评的 Agent 产品"
